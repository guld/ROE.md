## How to Contribute

1. Open a **PR** for **Bugs & small fixes**
2. Start a Github discussion for **New features / architecture** 
3. Add an issue if you tested an untested AI model, `ROE.md` version, or programming language combination. Check first if someone else posted a similar issue. 


Remember, ROE.md should stay a single Markdown file.

## Before You PR 

- Test locally with your AI coding assistant 
- Run tests
- keep PRs focused on one thing per PR
- Describe what & why

## Vibe-Coded AI PRs

Build with OpenCode, Codex or Claude? **Mark it!**

Include in your PR:

- [ ] Mark as AI-assisted in PR title and description
- [ ] Note the defree of testing (untested / lightly tested / fully tested)
- [ ] include prompts or session logs if possible (these are super helpful!)
- [ ] Confirm

Be transparent about your PRs.

## Focus & Roadmap

  - Improving performance and compacting context during the growth step.
  
## Report a Vulnerability

Please report any security vulnerabilities with to this repo directly.

Reports without reproduction steps, demonstrated impact, and remediation advice will be deprioritized.
